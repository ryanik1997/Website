Uu tien: HIGH. Thoi luong: 1-2 session.

2.1. Siet rate limit + phat hien hanh vi bat thuong
     - content-sign hien: 45 req/phut/user, 120 req/phut/IP, TTL 90s.
       Giu nguyen cho media (1 bai listening can nhieu part), nhung:
     - Them DAILY QUOTA theo user: vi du 400 lan ky/ngay/user (nguoi hoc
       that hiem khi vuot; crawler vuot ngay). Luu dem vao bang
       content_access_log (da co san tu migration 019) — dem bang query
       count(*) where user_id + created_at > now()-interval '1 day',
       tu choi voi code RATE_LIMIT_DAILY.
     - Them canh bao: 1 cron (Supabase pg_cron hoac GitHub Action) chay
       moi gio, query user co so lan ky > nguong, gui email/Zalo cho
       admin. Crawl co tai khoan tra phi la kich ban thuc te nhat —
       phat hien som quan trong hon chan cung.

2.2. Khoa signed URL chat hon
     - Giam SIGN_TTL_SEC tu 90 xuong 60 (du cho browser bat dau tai;
       audio dang phat khong bi cat vi da mo stream).
     - Can nhac Supabase Storage "transformations + token" hoac doi
       sang download qua edge function proxy neu muon gan
       Content-Disposition + chan Range abuse. (Chi lam neu thay
       bi share URL — TTL 60s da lam URL share gan nhu vo dung.)

2.3. Chan farm account
     - Bat "Confirm email" + captcha (Cloudflare Turnstile) trong
       Supabase Auth settings — Supabase ho tro Turnstile native,
       chi can bat + them site key vao LoginPage.
     - Gioi han: 1 tai khoan free chi xem duoc demo => dong luc farm
       account thap san. Turnstile la lop chan bot dang ky hang loat.

2.4. RLS ra soat toan cuc (1 luot, co checklist)
     - Voi TUNG bang trong 19 migrations, tra loi: "anon doc duoc gi?
       authenticated doc duoc gi cua nguoi khac?"
     - Da tot: profiles, decks, cards, srs, writing_docs, ai_usage,
       mindmaps, exam_progress, checkin_days, payment_requests,
       content_access_log (own + admin).
     - Sua o Phase 1: reading/listening_exam_published.
     - Xac nhan them: reading_exam_images (009) co policy public read —
       kiem tra bang nay chua URL anh hay binary; neu chi la mapping
       id->storage path thi chap nhan, vi anh that da private.
     - Chay `select * from pg_policies where schemaname='public'` tren
       production sau khi push de doi chieu voi file migration (tranh
       drift giua local va prod).

2.5. Khong de lo qua kenh phu

Uu tien: MEDIUM-HIGH (bat rat nhanh, hieu qua cao). Thoi luong: < 1 gio.

5.1. Vercel Dashboard => Firewall:
     - Bat "Bot Protection" / "Attack Challenge Mode" khi thay bi cao
       diem (challenge moi request bang browser check).
     - Tao rule: challenge/deny theo User-Agent bot pho bien
       (python-requests, scrapy, curl, wget, HeadlessChrome) cho path
       /app/* va /assets/*. Luu y: chi la rao can dau, UA gia duoc.
     - Rate limit rule cua Vercel WAF: gioi han request/IP/phut cho
       /assets/* (chan tai bundle hang loat de diff-clone UI).
5.2. Neu can manh hon nua (giai doan sau): dat Cloudflare truoc Vercel
     (proxy DNS) de dung Bot Fight Mode + Turnstile + ASN block

     Danh doi: them 1 lop phuc tap, chi lam khi co bang chung bi crawl.
5.3. Giam sat: bat Vercel Analytics + Log Drains; dinh ky xem top IP,
     top path. Ket hop voi content_access_log (2.1) la du 2 mat tran:
