Có thể **không gửi PDF đề** — gửi **file TXT UTF-8** thay thế. App import ZIP chỉ cần `exam.json` + media; không bắt buộc PDF trong ZIP.

`Book {B} · Test {T} · examType ket|pet|fce|cae|cpe · audioFile listening.mp3 · nguồn: TXT|PDF`

### Nội dung file TXT đề (`exam-text.txt` / `Test_N_Listening.txt`)

- UTF-8, **1 test / 1 file** (không gộp Test 1+2)
- Giữ cấu trúc: Part 1 … Part N, instruction, câu hỏi, options A/B/C, form gap (label dòng)
- Có thể gộp answer key + audioscript **trong cùng 1 TXT** nếu có marker rõ:
  - `ANSWER KEY` / `Part 1` + dòng `1 A`
  - `AUDIOSCRIPT` / `Tapescript` + đoạn theo số câu
- **Không** nhét base64 / binary; không cần gửi `listening.mp3` cho AI

## ⚠ Gap-fill — đáp án kép (app chấm điểm)

App **hiểu** các dạng sau (gõ `8` hoặc `eight` đều đúng):

| Cách viết trong exam.json | Ví dụ |
|---------------------------|--------|
| Slash `/` hoặc `\|` trong `answer` | `"answer": "8/eight"` · `"could/may"` |
| `OR` / dấu phẩy | `"answer": "8 OR eight"` |
| Mảng `acceptableAnswers` | `"answer": "8", "acceptableAnswers": ["eight"]` |
| Kết hợp | `"answer": "8/eight", "acceptableAnswers": ["8", "eight"]` |

```json
{
  "number": 6,
  "type": "gap-fill",
  "prompt": "Time guests arrive:",
  "answer": "8/eight",
  "acceptableAnswers": ["8", "eight"],
  "wordLimit": 1,
  "explanation": "Theo Answer key: 8 / eight."
}
```

**Bắt buộc:**
- Bỏ ngoặc key: `"(August)"` → `"August"`; `(19)` → `"19"` hoặc `"19/nineteen"`
- `wordLimit`: ONE WORD → `1`; ONE WORD AND/OR A NUMBER → thường `1`–`2`
- **Không** ghi `"answer": "B (best)"` — chỉ `"B"`
- Review UI hiện **「8 hoặc eight」** (không nhầm thành option A/B)

> App nhận **`answer-key.pdf`** và **`answer-key.txt`** (ưu tiên `.txt` nếu cả hai có).

App import ZIP nhận các tên (không phân biệt hoa thường):

| File trong ZIP | App làm gì |
|----------------|------------|
| `answer-key.pdf / answer-key.txt` / `Answer Key.pdf` | Trích text → bổ sung `answer` nếu JSON thiếu; nếu có block **Audioscript** → gắn `ttsText` từng câu |
| `audioscript.txt` / `tapescript.txt` / `transcript.txt` | Parse script → `ttsText` theo số câu |
| (IELTS Listening) | **Không** auto transcript từ PDF — user bật AI khi xem cùng đề |

### PDF Answer Key

- **Có text layer** (bôi đen/copy được chữ) → app trích được.
- **Scan ảnh** → thêm `audioscript.txt` (UTF-8) vào ZIP; preview import sẽ cảnh báo nếu PDF không có chữ.

### Cấu trúc text (answer-key hoặc audioscript.txt)

**1) Answer key** (phần đầu):

```
Part 1
1 A
2 C
…
Part 2
6 August
```

**2) Audioscript** (phần sau — marker bắt buộc để app tách transcript):

```
AUDIOSCRIPT
Part 1
1
Woman: Shall we meet at the cinema this evening?
Man: Yes, at half past seven.
2
Boy: Where is my bag?
Girl: It's under the table.
```

Marker nhận diện: `Audioscript` / `Audio script` / `Tapescript` / `Transcript`.

Mỗi câu: dòng chỉ số `1` hoặc `Question 1`, rồi đoạn thoại (không gộp dòng đáp án ngắn `1 A`).

### Tuỳ chọn ghi sẵn trong exam.json

```json
{

  "type": "picture-mc",
```

## ⚠ JSON HỢP LỆ — BẮT BUỘC

`exam.json` phải **JSON.parse PASS** — một lỗi cú pháp → toàn bộ đề không import được.

### 1. Dấu `"` trong chuỗi

**Bên trong chuỗi KHÔNG được có dấu `"` ASCII trần** — escape `\"` hoặc bỏ/thay.

| SAI | ĐÚNG |
|-----|------|
| `"explanation": "... cụm "rather than" ..."` | `"... cụm rather than ..."` |
| `"answer": "B (best)"` | `"answer": "B"` |

### 2. Trường `answer`

| Loại | `answer` ĐÚNG | SAI |
|------|---------------|-----|
| multiple-choice / picture-mc | `"A"` / `"B"` / `"C"` (hoặc chữ thường) | `"B (best)"` |
| matching | một chữ `"A"` … `"H"` | `"Paragraph D"` |
| gap-fill | `"August"`, `"great name"`, `"8/eight"` | `"(great) name"` — bỏ ngoặc key |
| đáp án kép | `"could/may"` · `"8/eight"` · + `acceptableAnswers` | `"rings (or sugar)"` |

Biến thể answer key → **`answer` có `/`** và/hoặc **`acceptableAnswers: ["…"]`** (+ `explanation`).

### 3. Lỗi JSON khác — KHÔNG được phép

- Hai JSON dính `}{`
- Trailing comma, comment `//`, smart quotes
- `questions` ngoài `parts[]`

### 4. Audio

- **Một MP3** cho cả bài → `"audioFile": "listening.mp3"` trên **mỗi part**
- App dedupe — chỉ lưu 1 bản trong Dexie
- Audio **không dừng** khi chuyển Part (mọi level)
- `imageFile` / `audioFile` trong JSON = tên file thật trong ZIP
