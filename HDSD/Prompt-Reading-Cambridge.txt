## Schema exam.json (chung A2–C2)

```json
{
  "version": 1,
  "title": "KET A2 Reading — Book 1 — Test 1",
  "durationMinutes": 60,
  "bandHint": "A2 Key Reading",
  "examTrack": "cambridge",
  "cambridgeLevel": "a2",
  "parts": [ /* đủ số parts theo level */ ]
}
```

### Trường bắt buộc mỗi part
- `partNumber`, `rangeLabel`, `passageTitle` (tiêu đề mô tả, vd "Part 4 — The coldest place to live")
- `passage[]`: `{ "text": "..." }` hoặc `{ "label": "Angus", "text": "..." }` hoặc `{ "imageFile": "part1-q1.jpg" }`
- `questionGroups[]`: `range`, `instruction`, `type`, `questions[]`

## Answer Key trong ZIP (app import)

App chấp nhận **cùng cấp** `exam.json`:

| File | Ghi chú |
|------|---------|
| `answer-key.txt` | **Khuyến nghị** — UTF-8, dễ parse (ưu tiên hơn PDF scan) |
| `answer-key.pdf` | OK nếu có **text layer** (copy được chữ) |
| `Answer Key.pdf` / `answers.txt` / `answer_key.txt` | Tên alias được nhận |

**Format dòng** (mỗi đáp án 1 dòng):

```
1 A
2 C
12 TRUE
15 not given
19 B
25 August
```

- App **chỉ bổ sung** câu trong JSON đang `answer` trống — không ghi đè đáp án đã điền.
- Writing task (KET P6–7, PET P7–8, FCE P8–9, …) **không** lấy từ key.
- PDF scan không trích được chữ → xuất `answer-key.txt` từ key.

### ⚠ Quy tắc phân đoạn — BẮT BUỘC mọi đề A2–C2

**PDF có bao nhiêu đoạn → `passage[]` có bấy nhiêu block `{ "text": "..." }`.**

| PDF | JSON |
|-----|------|
| 5 đoạn thân bài | 5 block text riêng |
| Mỗi người 1 đoạn (Part 2) | 1 block `{ "label": "Tên", "text": "..." }` / người |
| 2 email (Part 5 KET) | 2 block text — email 1: gaps 25–27; email 2: gaps 28–30 |
| Đáp án kép Part 5 | `answer: "từ1/từ2"` khi answer key chấp nhận 2 từ (app chấm cả hai) |
| Tiêu đề bold + subtitle italic (Part 3…) | `passageTitle` + `passageSubtitle` — **không** gộp vào passage[] |

**SAI:** gộp nhiều đoạn PDF vào 1 `text` dùng `\n\n`
**ĐÚNG:** mỗi đoạn văn PDF = 1 phần tử `passage[]`

Ngoại lệ duy nhất: Part 4 gapped text nếu PDF in **một khối liền** (không xuống dòng giữa câu) → giữ 1 block, nhưng vẫn giữ marker `(19) ........`.

### Quy tắc câu hỏi
- `answer`: MC → **chỉ** `a`/`b`/`c`/`d` (1 ký tự, chữ thường) — KHÔNG `"c (or all)"`, `"A"`, ghi chú trong answer
- gap-fill → một từ; đáp án kép key → **`từ1/từ2`** hoặc **`từ1|từ2`** (app chấp nhận user gõ từ1 **hoặc** từ2)
- `explanation`: giải thích ngắn tiếng Việt — **KHÔNG** dùng dấu `"` ASCII bọc từ trong chuỗi (vd SAI: `"từ "directly""` → ĐÚNG: `từ directly` hoặc `'directly'`)
- Mỗi câu có `type` khớp `questionGroups[].type`
- MC phải có đủ `options` với `id` + `label`
- **Ảnh WebP:** `imageFile: "part1-q1.jpg"` + file `part1-q1.webp` trong ZIP = OK

### Gap-fill / open cloze — đáp án kép (Reading)

```json
{ "number": 25, "type": "gap-fill", "prompt": "…", "answer": "are/is", "explanation": "Key: are / is" }
```

- Slash `/` hoặc pipe `|` trong `answer` → chấm đúng nếu user gõ **một** trong các biến thể
- Bỏ ngoặc answer key: `"(the) park"` → thường ghi `"park"` hoặc theo policy đề (ONE WORD)

### ⚠ JSON hợp lệ — tóm tắt (mọi level)
1. `exam.json` phải parse được bằng `JSON.parse()` — 1 lỗi cú pháp = import fail toàn bộ
2. Không dấu `"` trần trong `explanation`/`prompt`/`text` — escape `\"` hoặc bỏ / dùng nháy đơn `'...'`
3. Không trailing comma, comment `//`, smart quotes `"` `"`
4. Tự kiểm tra trước khi xuất ZIP

## Quy tắc bắt buộc
1. Đủ câu theo bảng level — đánh số liên tục, không nhảy số
2. Chỉ dùng nội dung PDF đề + Answer Key — KHÔNG bịa
3. Thiếu đáp án → `answer: ""` + `explanation: "⚠ Cần điền từ answer key"` (hoặc kèm `answer-key.txt` trong ZIP để app bổ sung)
4. `passageTitle` = tiêu đề mô tả (không dùng "PART X — QUESTIONS …" làm title duy nhất)
5. Part 1 A2: ưu tiên ảnh từng sign; Part 2+ bắt buộc text

## Việc làm theo thứ tự
1. Xác định level (a2/b1/b2/c1/c2) → đếm parts/câu
2. Đọc PDF → passage + câu hỏi + options
3. Đọc Answer Key (PDF text **hoặc** `.txt`) → answer + explanation
4. Validate JSON + tên ảnh khớp imageFile
5. Xuất ZIP: `exam.json` + ảnh + tuỳ chọn `answer-key.pdf` **hoặc** `answer-key.txt`
6. Bảng tóm tắt parts

Bắt đầu xử lý file đính kèm.
